Conceito 01
O que é uma Função?
☕ Contexto: pedido no café

Imagine que você entra no mesmo café toda manhã e pede: "Um café médio, com leite, sem açúcar." Cansativo repetir isso em detalhes a cada vez.

O atendente memoriza seu pedido e só precisa ouvir o nome. Uma função em Python funciona assim — você dá nome a uma sequência de passos e a chama quando precisar, quantas vezes quiser.

1
Escreva uma vez

O bloco def registra a função na memória do Python.

2
Chame quando precisar

Basta usar o nome — Python executa tudo dentro dela.

3
Adapte com parâmetros

Café com leite, sem açúcar, tamanho grande — cada chamada pode ser diferente.

⚠️
Definir ≠ Executar

O bloco def apenas registra. O código só roda quando você chama a função pelo nome com ().

cafe.py
# ── Sem função: código repetido ──────────
print("--- Pedido ---")
print("Café médio")
print("Com leite, sem açúcar")
# amanhã, terça, quarta... repete tudo

# ── Com função: escreve uma vez ──────────
def fazer_pedido():
    print("--- Pedido ---")
    print("Café médio")
    print("Com leite, sem açúcar")

# Chama quantas vezes quiser
fazer_pedido()   # segunda
fazer_pedido()   # terça
fazer_pedido()   # quarta...
▶ saída (cada chamada)--- Pedido --- Café médio Com leite, sem açúcar
💡
Nomes descritivos são documentação

fazer_pedido() diz o que faz sem precisar de comentário. Escolha nomes com verbos: calcular_, verificar_, exibir_.

Conceito 02
Por que usar funções?
🛒 Contexto: sistema de caixa de mercado
♻️
Reutilização

Escreva uma vez, use em qualquer parte do programa. Exibir um produto no caixa não precisa existir 50 vezes — apenas uma função.

📦
Organização

Em vez de 300 linhas seguidas, você tem calcular_preco(), aplicar_desconto() e emitir_nota() — cada uma com sua responsabilidade.

🔧
Manutenção

Corrigiu o cálculo de troco? Mude uma vez. Sem funções, você precisaria encontrar e corrigir cada cópia espalhada pelo código.

🔬
Testabilidade

Funções pequenas e isoladas são fáceis de testar. Você consegue verificar calcular_troco() sem executar o programa inteiro.

❌ Sem funções — lógica duplicada
# Produto 1
print("Café")
print(f"R$ 5.50 x 1 = R$ 5.50")

# Produto 2 — mesma estrutura!
print("Pão")
print(f"R$ 3.00 x 2 = R$ 6.00")

# Bug no formato? Corrigir em cada item.
✅ Com função — um lugar só
def exibir_item(nome, preco, qtd):
    subtotal = preco * qtd
    print(f"{nome}")
    print(f"R$ {preco:.2f} x {qtd} = R$ {subtotal:.2f}")

exibir_item("Café",  5.50, 1)
exibir_item("Pão",   3.00, 2)
exibir_item("Suco",  7.00, 1)
# Bug? Corrija só na função.
Conceito 03
Declarar e Chamar
📱 Contexto: app de notificações

Todo app envia notificações. Sempre o mesmo formato — mas com conteúdo diferente. Uma função define o molde; a chamada preenche com dados reais.

notificacoes.py
# ── 1. DECLARAÇÃO ───────────────────────
def notificar(app, mensagem):
    print("┌─────────────────────────┐")
    print(f"│ 🔔 {app:22} │")
    print(f"│ {mensagem:24} │")
    print("└─────────────────────────┘")

# ── 2. CHAMADAS ─────────────────────────
notificar("WhatsApp", "Ana: Oi, tudo bem?")
notificar("iFood",    "Pedido saiu p/ entrega")
notificar("Nubank",   "Pix recebido: R$50,00")
▶ saída┌─────────────────────────┐ │ 🔔 WhatsApp │ │ Ana: Oi, tudo bem? │ └─────────────────────────┘
Anatomia da Declaração
PartePapel
defPalavra reservada — inicia a declaração
notificarNome da função (snake_case, verbo)
(app, msg)Parâmetros — dados de entrada
:Abre o bloco
indentação4 espaços definem o corpo
🎯
Convenção de nomes

Use snake_case e comece com um verbo: calcular_, verificar_, exibir_, enviar_. Um bom nome dispensa comentário.

🔍
Onde declarar?

Sempre antes de chamar. Python lê o arquivo de cima para baixo; chamar uma função antes de declará-la gera NameError.

Conceito 04
Parâmetros e Argumentos
🧾 Contexto: recibo de compra online

Quando você compra online, o e-mail de confirmação tem sempre o mesmo layout — mas com seus dados. A função é o molde; os parâmetros são os espaços em branco.

Parâmetro

Variável na definição da função. Só existe enquanto ela executa. Recebe o valor passado na chamada.

def recibo(produto, preco, qtd):
Argumento

O valor concreto passado na chamada. Cada argumento é atribuído ao parâmetro correspondente.

recibo("Tênis", 299.90, 1)
🔑
Argumento nomeado

Passe em qualquer ordem usando o nome do parâmetro: recibo(qtd=2, preco=50, produto="Meia"). Torna chamadas longas muito mais legíveis.

loja.py
def emitir_recibo(produto, preco, qtd):
    subtotal = preco * qtd
    print("══════════════════════════")
    print(f"Produto : {produto}")
    print(f"Preço   : R$ {preco:.2f}")
    print(f"Qtd     : {qtd}x")
    print(f"Subtotal: R$ {subtotal:.2f}")
    print("══════════════════════════")

# Posicional — ordem importa
emitir_recibo("Tênis Nike", 299.90, 1)

# Nomeado — ordem livre
emitir_recibo(
    qtd=3,
    produto="Meia",
    preco=12.50
)
▶ saída (1ª chamada)══════════════════════════ Produto : Tênis Nike Preço : R$ 299.90 Qtd : 1x Subtotal: R$ 299.90 ══════════════════════════
❌
Erro comum: número errado de argumentos

Chamar emitir_recibo("Tênis") sem todos os parâmetros gera TypeError. Parâmetros obrigatórios exigem argumento.

Conceito 05
Valor Padrão
🍕 Contexto: app de delivery de pizza

A maioria dos pedidos é tamanho médio, sem borda recheada. Faz sentido esses serem os valores padrão — o cliente só informa quando quer algo diferente.

Um parâmetro com valor padrão (default) é opcional na chamada. Se nada for passado, Python usa o valor definido.

⚠️
Regra de ordem — obrigatória

Parâmetros com padrão sempre devem vir depois dos sem padrão.

❌ def f(a=1, b):
✅ def f(b, a=1):
🐛
Armadilha: listas como padrão

Nunca use def f(itens=[]). A lista é criada uma única vez e compartilhada entre todas as chamadas. Use itens=None e crie a lista dentro.

delivery.py
def fazer_pedido(
    sabor,                   # obrigatório
    tamanho="média",         # opcional
    borda="simples",         # opcional
    entrega=True             # opcional
):
    print(f"🍕 {sabor} | {tamanho}"
          f" | borda: {borda}"
          f" | entrega: {'sim' if entrega else 'não'}")

# Só o obrigatório — usa todos os padrões
fazer_pedido("Calabresa")

# Personaliza tamanho, mantém o resto
fazer_pedido("Frango", "grande")

# Mistura posicional + nomeado
fazer_pedido(
    "4 Queijos",
    tamanho="família",
    borda="catupiry",
    entrega=False
)
▶ saída🍕 Calabresa | média | borda: simples | entrega: sim 🍕 Frango | grande | borda: simples | entrega: sim 🍕 4 Queijos | família | borda: catupiry | entrega: não
Conceito 06
Retornando Valores
💰 Contexto: simulador de financiamento

Uma função que só imprime é como um caixa de banco que faz a conta mas não te mostra o resultado. Com return, a função devolve um valor que você pode usar, guardar ou passar adiante.

O que return faz
1
Devolve um valor

O resultado fica disponível para quem chamou a função.

2
Encerra a função imediatamente

Nenhuma linha após o return é executada.

3
Pode retornar múltiplos valores

return a, b, c cria uma tupla implícita.

⚠️
Sem return → devolve None

Guardar o resultado de uma função sem return em uma variável é um bug silencioso muito comum.

financiamento.py
def calcular_parcela(valor, meses, taxa_mes):
    # Fórmula de financiamento Price
    t = taxa_mes / 100
    fator = (t * (1 + t) ** meses) / \
            ((1 + t) ** meses - 1)
    parcela = valor * fator
    total   = parcela * meses
    juros   = total - valor
    return parcela, total, juros  # tupla

# Desempacotamento direto
p, t, j = calcular_parcela(
    valor=20_000,
    meses=48,
    taxa_mes=1.2
)
print(f"Parcela : R$ {p:,.2f}")
print(f"Total   : R$ {t:,.2f}")
print(f"Juros   : R$ {j:,.2f}")

# Retorno também pode alimentar outra função
print(round(calcular_parcela(5000,12,2.5)[0], 2))
▶ saídaParcela : R$ 527,03 Total : R$ 25.297,62 Juros : R$ 5.297,62 476.69
Conceito 07
Escopo de Variáveis
🏦 Contexto: conta bancária digital

Imagine uma agência bancária: o sistema central tem acesso ao saldo de todos os clientes (global), mas cada caixa numa cabine separada só enxerga os papéis na sua mesa. Quando o caixa vai embora, esses papéis somem.

🌐 Escopo Global

Variáveis declaradas fora de funções. Acessíveis em todo o arquivo. Vivem enquanto o programa rodar.

🔒 Escopo Local

Variáveis criadas dentro de uma função — incluindo os parâmetros. Invisíveis fora dela. Destruídas quando a função termina.

⚠️
global — use com cautela

global saldo permite modificar uma variável global de dentro de uma função. Funciona, mas dificulta rastrear bugs. Prefira passar e retornar valores.

banco.py
saldo = 1_000.00   # ← GLOBAL

def depositar(valor):
    taxa  = 0.005    # ← LOCAL (só existe aqui)
    bonus = valor * taxa   # ← LOCAL
    return valor + bonus  # devolve, não modifica global

def sacar(valor, saldo_atual):
    if valor > saldo_atual:
        print("❌ Saldo insuficiente")
        return saldo_atual
    return saldo_atual - valor

# ── Boas práticas: passa e recebe ───────
saldo = depositar(500.00)      # → 1502.50
saldo = sacar(200.00, saldo)    # → 1302.50
print(f"Saldo: R$ {saldo:.2f}")

# ── Isso causaria NameError: ─────────────
# print(taxa)   ← local de depositar()
# print(bonus)  ← local de depositar()
▶ saídaSaldo: R$ 1302.50
❌
Armadilha: ler global sem querer

Se você digitar errado um parâmetro, Python pode ler a variável global de mesmo nome sem avisar. Nomes diferentes para globais e locais evitam esse bug.

Conceito 08
Composição de Funções
🚗 Contexto: app de corridas (Uber / 99)

O preço de uma corrida envolve várias etapas: distância, tempo, tarifa dinâmica, taxa do app, gorjeta. Cada responsabilidade vira uma função. A função principal apenas orquestra as demais.

Isso é composição de funções — a base da programação modular.

Por que compor?
▸
Separação de responsabilidade

Cada função faz uma coisa e a faz bem.

▸
Testabilidade

Você testa calcular_base() isoladamente, sem rodar o app todo.

▸
Reuso

aplicar_taxa() pode ser usada em outros contextos do sistema.

💡
Princípio da Responsabilidade Única

Se você precisa de "e" para descrever o que a função faz, considere dividi-la em duas.

corrida.py
def calcular_base(km, minutos):
    return 2.50 + km * 1.80 + minutos * 0.35

def aplicar_pico(preco, fator=1.8):
    # fator > 1 em horário de pico
    return preco * fator

def adicionar_gorjeta(preco, pct=10):
    return preco * (1 + pct / 100)

def calcular_corrida(km, minutos,
                      pico=False, gorjeta=0):
    preco = calcular_base(km, minutos)  # chama 1
    if pico:
        preco = aplicar_pico(preco)     # chama 2
    if gorjeta > 0:
        preco = adicionar_gorjeta(preco, gorjeta) # chama 3
    return round(preco, 2)

# Corrida normal (8 km, 15 min)
print(calcular_corrida(8, 15))

# Horário de pico + gorjeta 10%
print(calcular_corrida(8, 15,
      pico=True, gorjeta=10))
▶ saída24.65 48.78

Exercícios Práticos

Resolva cada desafio antes de avançar. Comece pelos obrigatórios — o desafio é para quem terminar cedo.

01 — Gorjeta

Crie gorjeta(valor, pct=10) que retorne uma tupla com o valor da gorjeta e o total a pagar.

gorjeta(85.00) → (8.5, 93.5) gorjeta(200, pct=15) → (30.0, 230.0)
02 — Senha Válida

Crie senha_valida(s, minimo=8) retornando True se a senha tiver ao menos minimo caracteres e pelo menos um dígito.

senha_valida("abc") → False senha_valida("senha2024") → True
03 — Conversor

Crie converter(valor, de, para) para converter entre "km", "m", "cm" e "mm" usando um dicionário de fatores.

converter(1.5, "km", "m") → 1500.0 converter(200, "cm", "m") → 2.0
04 — IMC

Crie calcular_imc(peso, alt) e classificar_imc(imc). Depois, relatorio(nome, peso, alt) chama as duas e exibe um resumo.

relatorio("Ana", 62, 1.65) → Ana | 22.8 | Peso normal
🔥 Desafio — Mini Extrato Bancário

Implemente depositar(v, s), sacar(v, s) e extrato(operacoes). O parâmetro operacoes é uma lista de tuplas ("D", 200) ou ("S", 50). A função extrato deve iterar, chamar as demais e exibir cada linha com saldo parcial e saldo final.

extrato([("D",500), ("S",120), ("D",80), ("S",600)]) Depósito +R$500.00 → Saldo: R$500.00 Saque -R$120.00 → Saldo: R$380.00 Depósito + R$80.00 → Saldo: R$460.00 ❌ Saldo insuficiente para R$600.00 ────────────────────────────────────── Saldo final: R$460.00
