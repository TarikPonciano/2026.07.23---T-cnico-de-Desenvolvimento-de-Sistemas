Como fazer o computador repetir uma tarefa automaticamente — sem copiar e colar código dezenas de vezes.

for while break continue range()
Motivação
Senac TDS · Python

Por que precisamos de repetição?

Imagine que você precisa exibir a tabuada do 7. Sem repetição, o código ficaria assim:

print("7 x 1 =", 7*1)
print("7 x 2 =", 7*2)
print("7 x 3 =", 7*3)
... 7 linhas repetidas manualmente
print("7 x 10 =", 7*10)

E se fossem 100 linhas? Ou 1.000.000? Com estruturas de repetição (loops), o mesmo resultado em 2 linhas:

for i in range(1, 11):
    print(f"7 x {i} = {7*i}")
Pense em um loop como uma máquina de lavar roupa. Você não lava cada peça manualmente — você define o programa (quantas voltas, temperatura) e a máquina repete o ciclo sozinha até terminar.

Dois tipos de loop em Python

  • for — repete um número conhecido de vezes ("faça isso 10 vezes")
  • while — repete enquanto uma condição for verdadeira ("continue enquanto…")
Loops são um dos conceitos mais poderosos da programação. Dominar o for e o while é dominar a automação.
Loop for — Parte 1

O loop for e o range()

O for percorre uma sequência de valores. A função range() gera essa sequência de números inteiros.

for variável in range(início, fim, passo):
    # bloco indentado — repete a cada volta
    instrução
O for é como um funcionário que recebe uma lista de tarefas numeradas: "Faça a tarefa 1, depois a 2, depois a 3… pare no 10." Ele sabe exatamente quantas voltas vai dar.

range() — 3 formas de usar

range(5)          # 0, 1, 2, 3, 4
range(1, 6)       # 1, 2, 3, 4, 5
range(0, 11, 2)  # 0, 2, 4, 6, 8, 10
range(10, 0, -1) # 10, 9, 8... 1 (decrescente)

O fim é exclusivo — range(1,6) vai até 5, não chega ao 6.

Exemplos Práticos

# Contar de 1 a 5
for i in range(1, 6):
    print(i)
# saída: 1  2  3  4  5

# Tabuada do 9
for n in range(1, 11):
    print(f"9 x {n} = {9*n}")

# Contagem regressiva (lançamento)
for t in range(10, 0, -1):
    print(f"{t}...")
print("🚀 Lançamento!")

# Soma de 1 a 100
total = 0
for i in range(1, 101):
    total += i  # total = total + i
print("Soma:", total)  # 5050
A indentação define o que está dentro do loop. Tudo com 4 espaços após o for repete. O que não tem indentação executa apenas uma vez, depois do loop.

Contadores e Acumuladores com for

Um dos usos mais comuns do loop for é acumular valores — somar, contar, calcular médias — usando uma variável que cresce a cada iteração.

Padrão Contador

# Conta quantos números pares há entre 1 e 20
contador = 0           # inicia em 0

for i in range(1, 21):
    if i % 2 == 0:
        contador += 1   # incrementa

print("Pares encontrados:", contador)
# saída: Pares encontrados: 10

Padrão Acumulador (soma)

# Média de N números digitados pelo usuário
n = int(input("Quantos números? "))
soma = 0

for i in range(1, n + 1):
    valor = float(input(f"Número {i}: "))
    soma += valor

media = soma / n
print(f"Média: {media:.2f}")
Um acumulador é como um caixa de supermercado que vai somando o preço de cada produto na medida em que passa pelo leitor. Começa em zero e vai crescendo a cada item (iteração).

Operadores de atribuição

x += 1   # x = x + 1   (incremento)
x -= 1   # x = x - 1   (decremento)
x *= 2   # x = x * 2   (dobrar)
x /= 2   # x = x / 2   (dividir)
x **= 2  # x = x ** 2  (elevar ao quadrado)

Produto fatorial (n!)

n = int(input("Calcular fatorial de: "))
fatorial = 1           # neutro da multiplicação

for i in range(1, n + 1):
    fatorial *= i

print(f"{n}! = {fatorial}")

For Aninhado — loop dentro de loop

Assim como o if pode ser aninhado, podemos colocar um for dentro de outro for. Para cada iteração do loop externo, o loop interno executa completamente.

Imagine um relógio: o ponteiro dos segundos (loop interno) dá 60 voltas completas para cada volta do ponteiro dos minutos (loop externo). Os dois giram juntos, em níveis diferentes. 
# Tabuada de 1 a 5
for tabela in range(1, 6):
    print(f"\n=== Tabuada do {tabela} ===")
    for mult in range(1, 11):   # loop interno
        print(f"  {tabela} x {mult} = {tabela*mult}")

O loop externo itera 5 vezes. O interno, 10 vezes. Total: 50 linhas geradas automaticamente.

Padrão com estrelas — visualizando a iteração

# Imprime um triângulo de estrelas
for linha in range(1, 6):
    for col in range(linha):
        print("*", end="")
    print()  # quebra de linha
*
**

*****
Loops aninhados multiplicam as execuções. Um for de 100 dentro de um for de 100 executa 10.000 vezes. Use com consciência.

O loop while

O while repete um bloco enquanto uma condição for verdadeira. Diferente do for, ele não sabe de antemão quantas vezes vai repetir — depende do que acontecer durante a execução.

while condição:
    # bloco executado enquanto True
    instrução_1
    instrução_2
    # algo que eventualmente torna condição False
O while é como um semáforo de pedestre: enquanto a luz estiver vermelha (condição verdadeira), você fica parado repetindo a espera. Assim que fica verde (condição falsa), você atravessa e o loop termina.
Loop infinito! Se a condição nunca se tornar falsa, o programa trava para sempre. Sempre garanta que algo dentro do bloco modifique a condição.

Exemplo — Adivinhe o número

secreto = 42
chute   = 0        # valor inicial ≠ secreto

while chute != secreto:
    chute = int(input("Adivinhe (1-100): "))

    if chute < secreto:
        print("📉 Muito baixo!")
    elif chute > secreto:
        print("📈 Muito alto!")

print("🎉 Acertou!")

Aqui não sabemos quantas tentativas o usuário vai precisar — por isso o while é ideal. O for não serviria, pois não há um número fixo de repetições.

Quando usar while?

  • Número de repetições desconhecido
  • Depende de entrada do usuário
  • Espera por um evento ou condição externa

Padrões essenciais com while

Padrão 1 — Contador Manual

# Equivalente ao for range(1,6)
i = 1               # 1. inicialização

while i <= 5:       # 2. condição
    print(i)
    i += 1          # 3. atualização ⚠ obrigatória!

Padrão 2 — Validação de entrada

# Força o usuário a digitar valor válido
nota = -1          # valor inválido para entrar no loop

while nota < 0 or nota > 10:
    nota = float(input("Nota (0–10): "))
    if nota < 0 or nota > 10:
        print("❌ Nota inválida! Tente novamente.")

print(f"Nota registrada: {nota}")

Padrão 3 — Menu Interativo

opcao = ""

while opcao != "0":
    print("\n=== MENU ===")
    print("1 - Nova venda")
    print("2 - Ver relatório")
    print("0 - Sair")
    opcao = input("Escolha: ")

    if opcao == "1":
        print("Registrando venda...")
    elif opcao == "2":
        print("Exibindo relatório...")

print("Sistema encerrado.")
O padrão de menu com while é como um restaurante: o garçom continua tomando pedidos até o cliente dizer "a conta, por favor" (opção 0). O loop só encerra quando você decide sair.

break e continue — controlando o loop

break — Interrompe o loop

O break encerra imediatamente o loop, mesmo que a condição ainda seja verdadeira ou que haja iterações restantes.

# Procura o primeiro múltiplo de 7 entre 1 e 50
for n in range(1, 51):
    if n % 7 == 0:
        print(f"Primeiro múltiplo: {n}")
        break           # sai do loop aqui
# saída: Primeiro múltiplo: 7
O break é como o botão de emergência de uma esteira: assim que você o pressiona, tudo para imediatamente, sem esperar o ciclo atual terminar.

continue — Pula para a próxima iteração

O continue ignora o restante do bloco naquela iteração e volta ao início do loop para a próxima.

# Imprime só os ímpares de 1 a 10
for i in range(1, 11):
    if i % 2 == 0:
        continue        # pula os pares
    print(i)
# saída: 1 3 5 7 9
O continue é como um portão seletivo numa linha de produção: quando um produto com defeito passa, o portão o desvia e o processo segue para o próximo — sem parar a linha inteira.
break encerra o loop inteiro. continue encerra apenas a iteração atual e vai para a próxima.
Padrão Avançado
Senac TDS · Python

O padrão while True

O while True cria deliberadamente um loop infinito, que só termina quando um break é encontrado. É muito usado em menus e sistemas de login.

SENHA_CORRETA = "python123"
tentativas = 0

while True:
    senha = input("Digite a senha: ")
    tentativas += 1

    if senha == SENHA_CORRETA:
        print("✅ Acesso liberado!")
        break           # sai do while True
    elif tentativas >= 3:
        print("🔒 Conta bloqueada!")
        break
    else:
        restantes = 3 - tentativas
        print(f"❌ Senha errada. {restantes} tentativa(s).")
O while True é como uma catraca automática: ela fica girando indefinidamente, esperando o cartão certo. Quando o cartão correto é passado (break), a catraca para. Se errar 3 vezes, o segurança bloqueia a entrada (segundo break).

Comparando os dois padrões

# Padrão tradicional
continua = True
while continua:
    r = input("Continua? (s/n): ")
    if r == "n":
        continua = False

# Padrão while True + break (mais limpo)
while True:
    r = input("Continua? (s/n): ")
    if r == "n":
        break

for vs while — quando usar cada um?

Critério for while
Nº de repetições Conhecido antes de começar Desconhecido — depende da condição
Controle Automático via range() Manual — você gerencia o contador
Risco de loop infinito Baixo — range() sempre termina Alto — se a condição nunca for falsa
Uso típico Tabuadas, somas, contadores, padrões Menus, validação, login, jogos
Metáfora Máquina de lavar: ciclo definido Semáforo: repete até mudar de cor

Prefira o for quando…

  • Você sabe exatamente quantas vezes repetir
  • Quer percorrer um intervalo numérico (range)
  • Está calculando somas, contadores, fatoriais

Prefira o while quando…

  • A repetição depende de input do usuário
  • Você cria menus ou sistemas interativos
  • Precisa validar dados até obter uma resposta correta

Caixa Eletrônico — Projeto Completo

Vamos integrar for, while, if/elif/else, break e acumuladores num programa real.

saldo     = 1000.0
tentativas = 0
MAX       = 3

# ── Autenticação ──────────────────────
while tentativas < MAX:
    pin = input("Digite seu PIN: ")
    if pin == "1234":
        print("✅ Autenticado!")
        break
    tentativas += 1
    print(f"PIN errado. {MAX - tentativas} tentativa(s).")
else:              # else do while: executa se NÃO houve break
    print("🔒 Cartão bloqueado.")
    exit()

# ── Menu principal ────────────────────
while True:
    print(f"\nSaldo: R$ {saldo:.2f}")
    print("1-Saque  2-Depósito  0-Sair")
    op = input("Opção: ")

    if op == "0":
        print("Obrigado. Até logo!")
        break
    elif op == "1":
        v = float(input("Valor: R$ "))
        if v > saldo:
            print("❌ Saldo insuficiente!")
        else:
            saldo -= v
            print(f"💸 Saque de R$ {v:.2f} realizado.")
    elif op == "2":
        v = float(input("Valor: R$ "))
        saldo += v
        print(f"💰 Depósito de R$ {v:.2f} realizado.")

O que este programa usa

  • while com contador para autenticação (máx. 3 tentativas)
  • break para sair quando PIN correto
  • else do while — executa se não houve break
  • while True para o menu principal
  • if/elif/else para as operações
  • Acumulador com -= e += para o saldo

💡 else no while

Python tem um recurso raro: o bloco else depois de um while ou for. Ele executa somente se o loop terminou normalmente — sem break. Muito útil para sinalizar "procurei e não achei".
